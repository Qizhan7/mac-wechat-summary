============================================
    微信群聊 AI 总结 - 功能说明
============================================

启动后菜单栏会出现 💬 图标（在屏幕上方），点击即可操作。
以下介绍所有功能和使用技巧。



============================================
    一、群聊总结（核心功能）
============================================

每个群聊点开后有两个选项：

  📝 总结新消息
  ─────────────
  自动读取「上次总结之后」的所有新消息，
  发送给 AI 生成总结，然后自动弹出总结文件。

  • 首次使用会读最近的消息（最多 500 条）
  • 之后每次只读「上次到现在」的新消息
  • 总结完成后自动标记已读位置（书签）

  🔧 自定义总结
  ─────────────
  不按书签，手动指定总结范围。两种方式二选一：

  • 输入条数：如 100 → 总结最近 100 条消息
  • 输入分钟数：如 30m → 总结最近 30 分钟内的消息

  适合回顾特定时间段、或跳过部分消息重新总结。



============================================
    二、分组管理
============================================

把常看的群聊归到一起，方便批量操作。

  新建分组
  ─────────────
  菜单底部有「📂 ✨ 新建分组…」，输入名称即可。
  比如：工作群、购物群、学习群。

  添加群聊到分组
  ─────────────
  点开某个分组 → 「➕ 添加群聊…」
  会列出所有可添加的群聊，输入序号即可，
  多个用逗号分隔（如 1,3,5）。

  从分组移除
  ─────────────
  在分组内某个群聊的子菜单点「❌ 从分组移除」。

  删除分组
  ─────────────
  在分组子菜单底部点「🗑️ 删除分组」。
  仅移除分组，不影响群聊本身和历史总结。

  🚀 一键总结（分组批量总结）
  ─────────────
  在分组子菜单中点「🚀 一键总结」，
  会把该分组内所有群聊的新消息汇总到一份报告中。
  适合每天快速过一遍多个群的动态。



============================================
    三、关键词搜索
============================================

点击菜单中的「🔍 关键词搜索」进入搜索。

  搜索条件
  ─────────────
  • 关键词：多个用空格分隔，必须全部同时出现才命中
  • 开始日期：必填，格式 YYYY-MM-DD（如 2026-03-01）
  • 结束日期：选填，留空则为今天
  • 群聊范围：默认「全部」，也可输入序号指定
  • AI 总结：勾选后用 AI 整理搜索结果；不勾选则显示原文

  ⚠ 注意
  ─────────────
  • 开始日期是必填的，格式必须是 YYYY-MM-DD
  • 留意对话框底部的「用 AI 总结搜索结果」复选框，
    勾上会调用 AI 对搜到的消息做归纳；不勾则直接列出原文
  • 搜索是只读的，不会影响任何书签或已读状态



============================================
    四、总结记录
============================================

  菜单中的显示
  ─────────────
  • 📝 最新的一条总结会直接显示在菜单中，
    展开可以预览前几行、复制到剪贴板、或打开完整文件
  • 📋 最近总结：列出最近 15 条历史总结，点击即可查看

  文件存储
  ─────────────
  所有总结都保存在 ~/.wechat-summary/ 目录下的 .txt 文件中，
  文件名包含群名和时间戳，可以随时翻阅。
  也可以在菜单中点「📁 打开总结目录」直接打开。



============================================
    五、AI 服务
============================================

支持以下 AI 服务，在设置中切换：

  • 通义千问（默认） — 阿里云，国内访问快（本人只验证过这个，强烈推荐，免费额度多，只需要申请一个key）
  • DeepSeek         — 性价比高
  • Ollama 本地      — 完全离线，无需 Key
  • Claude           — Anthropic
  • OpenAI           — GPT 系列

切换 AI 服务后，需要设置对应的 API Key。
Key 安全存储在 macOS 钥匙串中，不会明文保存到文件。



============================================
    八、注意事项和使用技巧
============================================

  群聊列表刷新
  ─────────────
  • 推荐手动刷新：点击菜单顶部的「刷新群聊列表」
  • 也可以在设置中开启「自动刷新」，打开菜单时自动更新，
    但会有一点延迟（需要等几秒列表才出来）
  • 默认显示最近的 50 个群聊

  什么时候要「重新提取密钥」
  ─────────────
  • 微信更新后，启动脚本会自动检测并处理
  • 如果看到通知「发现 X 个新数据库」，
    说明微信新增了数据库文件，点菜单中的「重新提取密钥」即可
  • 重新登录微信账号后，也需要重新提取
  • 提取密钥需要输入电脑密码（管理员权限）

  书签机制（有时会有几条的误差，手动刷新群聊列表）
  ─────────────
  • 每次总结后，程序会自动记录「读到哪里了」（书签）
  • 下次点「📝 总结新消息」时只读新增部分，不会重复
  • 菜单中每个群聊会显示上次总结时间和新消息数量
  • 如果想重新来过，去设置中点「重置所有书签」

  自定义总结的使用场景
  ─────────────
  • 想回顾更早的消息：用条数，如输入 200
  • 想看某个时间段：用分钟数，如 60m = 最近一小时
  • 自定义总结不受书签限制，可以随时回看

  显示群昵称（备注名）
  ─────────────
  • 在设置中可以开关「总结中显示群昵称」（默认开启）
  • 开启后，总结中的群成员名字会优先使用你设置的备注名
  • 优先级：你的备注名 > 群名片 > 微信昵称
  • 适合群成员多、昵称难辨认的群聊



============================================
    七、MCP Server（高级功能）
============================================

支持通过 Claude Desktop 或 Claude Code 直接查询和总结微信消息。

  配置方法
  ─────────────
  在 Claude Desktop 配置文件中添加：
  ~/Library/Application Support/Claude/claude_desktop_config.json

  {
    "mcpServers": {
      "wechat-summary": {
        "command": "/项目路径/.venv/bin/python3",
        "args": ["/项目路径/mcp_server.py"]
      }
    }
  }

  配置完成后，可以在 Claude 对话中直接：
  • 查询群聊列表
  • 读取聊天记录
  • 让 AI 总结指定群聊的消息
  • 搜索关键词



============================================
    八、注意事项和使用技巧
============================================

  关于数据安全
  ─────────────
  • 程序只读取微信数据库的副本，不会修改微信的任何文件
  • 消息内容只发送给你选择的 AI 服务，不经过其他服务器
  • 如果使用 Ollama 本地模型，消息完全不会离开你的电脑
  • API Key 存储在 macOS 钥匙串中，受系统级加密保护
